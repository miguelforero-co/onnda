<threat_model>
## Trust Boundaries
| Boundary | Description |
|----------|-------------|
| frontend → plugin commands | dialog/fs/updater commands are gated by capabilities; missing entry = silent denial |
| network → updater | update manifest + bundle fetched over network, must be signature-verified |

## STRIDE Threat Register
| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-04 | Elevation | over-broad fs capability | mitigate | grant scoped `fs:allow-read-file`/`fs:allow-write-file` rather than blanket recursive write; data-mgmt uses app_data_dir paths only |
| T-01-05 | Tampering | unsigned/forged update bundle | mitigate | updater plugin verifies minisign signature against pubkey in tauri.conf.json (user_setup: keypair generation) — Phase-1 fallback is check-only if infra not ready (RESEARCH Open Q3) |
| T-01-06 | Spoofing | dialog opening arbitrary files | accept | dialog:allow-open returns a user-chosen path; the file is read by symphonia which bounds resource use (plan 01-05) |
</threat_model>
