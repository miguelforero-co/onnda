# Phase 1 — UI Design Contract

## Spacing Scale

Declared values (multiples of 4, anchored to existing rhythm — current code uses a 4/8/14/20/22px cadence; this contract normalizes to the 4-base scale while permitting the documented exceptions already shipping):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-label gaps, inline chip padding, knob inset |
| sm | 8px | Compact control gaps (toggle rows, badge groups, dl-progress) |

| 2xl | 48px | Empty-state vertical padding |

Exceptions (already in the shipping design system — preserve, do NOT "fix" to multiples of 4):

- Sidebar width `200px` (CONTEXT D-01), nav item height target `34px`.
- Toggle dimensions `36×20px`, knob `16px` (existing `.toggle`/`.knob`) — do not alter.

## Typography

Exactly TWO declared weights: **450 (regular / body) and 600 (semibold / emphasis)**. Display, Heading, and section Labels all use 600; Body and meta use 450. There are no intermediate weights — when the executor touches any existing 550/650/700 spot, it normalizes to 450 or 600.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Display (onboarding/empty hero) | 22px | 600 | 1.2 |
| Heading (page title per section) | 16px | 600 | 1.3 |
| Body / row label | 13px | 450 | 1.5 |
| Label / meta (section labels = 600 uppercase; meta = 450) | 11px | 600 / 450 | 1.5 |

Notes:
- Section labels: `11px`, `600`, `uppercase`, `letter-spacing:.06em`, color `--faint` (existing 10.5px shipping value normalizes to 11px).
- Heading 16px is NEW for this phase — each content section (Home, Transcripciones, Diccionario, Ajustes) gets a page title at 16px/600 to anchor the sidebar navigation.
- Body line-height 1.5 (matches existing `.hist-text`/`.section-hint`); display line-height 1.2; hero paragraph 1.7 is an accepted exception (existing `.ob-intro p`).

## Color

60/30/10 split, mapped onto existing tokens. The palette is LOCKED (D-04) — this section assigns roles, it does not introduce colors.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--bg` `#F4F0EB` | App background, content-area canvas, active-nav pill |
| Secondary (30%) | `--panel` `#FDFCFA` | Sidebar surface, setting cards (`.rows`), Home dashboard cards, list rows |
| Accent (10%) | `--coral` `#E85535` | See reserved-for list below |
| Destructive | `--coral` `#E85535` | Destructive actions reuse coral (no separate red in palette) — paired with a confirmation step; destructive *intent* signaled by copy + confirm, not a new hue |

Accent (`--coral`) reserved for — EXPLICIT list, never "all interactive elements":
1. Primary CTA fill (`.btn-primary`: record/dictar primary action on Home, onboarding continue, confirm buttons).
2. The single record/dictation affordance on Home (the hero action).
3. Toggle "on" track (`.toggle.on`) and range `accent-color`.
4. Text links / inline actions (`.link-btn`: "Descargar", "Abrir ajustes", "Agregar").
5. Active download progress bar fill (`.dl-bar`) and focus ring (`border-color:rgba(232,85,53,.4)`).
6. Permission/attention banner accents (`.banner` coral variant + `.banner-link`).

Supporting semantic colors (NOT accent — narrow, reserved use, already shipping):
- `--blue` `#7B9BD2`: success/positive status ONLY — "Instalado" badge, "Concedido"/granted permission dot+label, success banner.
- `--amber` `#F4AA6A`: transient active-playback state ONLY (`.icon-btn.active` while audio plays).

Coral must NOT bleed onto: inactive nav items, section labels, body text, card surfaces, or the "Próximamente" Parakeet card (which is muted/disabled).

## Interaction States — Key Capture (Hotkey Recorder, CONTEXT D-11)

The hotkey recorder replaces the current `<input type=text>`. Spec its states explicitly (executor source of truth):

| State | Visual | Behavior |
|-------|--------|----------|
| Idle | Chip showing current combo (e.g. `Alt + Space`) using existing `kbd` styling; row label "Atajo de teclado" | Click anywhere on the chip enters Capturing. |
| Capturing | Chip replaced by pulsing coral-bordered field, text "Presiona la combinación…" | Listens for keydown; ignores lone modifier presses; first non-modifier key + held modifiers = candidate. Escape cancels back to Idle (reuse existing global-Escape passive monitor pattern). |
| Validating | Brief inline check | Reject combos with no modifier or reserved system combos → show inline hint "Usa al menos un modificador (⌘/⌥/⌃)". |
| Saved | Returns to Idle with new combo chip | Persist via `schedSave(true)` (`shortcutChanged: true`). GOTCHA: re-entrant global shortcut registration can hang — re-register only after capture completes, never mid-capture. |

Model selector cards (CONTEXT D-13): card per Whisper model (base, small, medium, large-v3-turbo) showing name + size/version + state. States = Not downloaded ("Descargar" link), Downloading (existing `.dl-bar` + `%`), Installed (`--blue` "Instalado" badge), Selected (coral 1px ring on the card). Parakeet card = disabled "Próximamente".

Permissions panel (D-09): reuse onboarding `.perm-row` (dot + label + status/action), live polling via existing `checkPerms` (3s interval). Granted = `--blue` dot + "Concedido"; not granted = muted dot + "Abrir ajustes" link.
