  <action>
    El problema estructural: los `.expect()` de `audio.rs:73,75` viven dentro de `std::thread::spawn`. Un panic ahí NO se propaga a `start()` — mata el thread en silencio y `start()` devuelve `Ok` con un stream muerto. El objeto `cpal::Stream` debe permanecer en el thread (mantiene vivo el audio), así que NO se puede devolver al caller; se devuelve un `Result<(), String>` por canal.

    En `src-tauri/src/audio.rs`, dentro de `start()`:
    1. Antes de `std::thread::spawn`, crear el canal: `let (tx, rx) = std::sync::mpsc::channel::<Result<(), String>>();`
    2. Mover `tx` al closure del thread. Reemplazar `.expect("failed to build input stream")` (línea 73) por un match que mapee `cpal::BuildStreamError` a español y señale el resultado:
    ```rust
    let stream = match device.build_input_stream(
        &config.into(),
        move |data: &[f32], _| { /* … callback existente sin cambios … */ },
        |err| log::error!("[audio] cpal stream error: {err}"),   // antes: eprintln!("cpal error: {err}")
        None,
    ) {
        Ok(s) => { tx.send(Ok(())).ok(); s }
        Err(e) => {
            let msg = match e {
                cpal::BuildStreamError::DeviceNotAvailable =>
                    "El micrófono dejó de estar disponible. Revisa que no esté en uso por otra app.".to_string(),
                cpal::BuildStreamError::StreamConfigNotSupported =>
                    "El micrófono no admite la configuración solicitada.".to_string(),
                e => format!("Error al configurar el micrófono: {e}"),
            };
            tx.send(Err(msg)).ok();
            return;
        }
    };
    ```
    3. Reemplazar `stream.play().expect("failed to start stream")` (línea 75) por:
    ```rust
    if let Err(e) = stream.play() {
        log::error!("[audio] stream.play() error: {e}");
        return;
    }
    ```
    4. Tras el `thread::spawn`, ANTES de `Ok(Self { ... })`, bloquear brevemente para recibir el resultado de construcción del stream:
    ```rust
    rx.recv()
        .map_err(|_| anyhow!("El thread de audio terminó inesperadamente"))?
        .map_err(|e| anyhow!("{e}"))?;
    ```
    Mantener el resto de `start()` (device/config/samples/stop/last_emit/fft, el `while !stop_thread...`, y el `Ok(Self {...})` final) sin cambios. El error callback de la línea 70 queda convertido a `log::error!` en el paso 2.
  </action>
