macOS impl pattern (mirror mic_permission.rs L23-29):
```rust
#[cfg(target_os = "macos")]
fn play_named(name: &str) {
    use objc2::runtime::AnyClass;
    use objc2::msg_send;
    use objc2_foundation::NSString;
    unsafe {
        let Some(cls) = AnyClass::get(c"NSSound") else { return };
        let ns_name = NSString::from_str(name);
        let sound: *mut objc2::runtime::AnyObject = msg_send![cls, soundNamed: &*ns_name];
        if sound.is_null() { return; }
        let _: bool = msg_send![sound, play];
    }
}
#[cfg(target_os = "macos")]
pub fn play_listen() { play_named("Tink"); }
#[cfg(target_os = "macos")]
pub fn play_stop()   { play_named("Pop"); }
#[cfg(target_os = "macos")]
pub fn play_cancel() { play_named("Funk"); }
```
Add the no-op twin (escape.rs L44-45 style):

```rust
#[cfg(not(target_os = "macos"))]
pub fn play_stop() {}
#[cfg(not(target_os = "macos"))]
pub fn play_cancel() {}
```

Implement `src-tauri/src/media_pause.rs` exposing `pause_if_playing()` and `resume_if_paused()`. Use a module-level `static I_PAUSED: AtomicBool = AtomicBool::new(false)` so resume only re-sends when WE initiated the pause (RESEARCH Pitfall 4). Send the Play/Pause media key via a `CGEvent` of type `NSSystemDefined` (event type 14), subtype 8, key `NX_KEYTYPE_PLAY` (16), down then up.

Mirror the FFI shape from `post_cmd_v` (L605-633) but for a system-defined event. Since constructing an `NSEvent.otherEventWithType:` system-defined event via objc2 is the reliable path, build it with objc2-app-kit `NSEvent` (already a dep feature) — `otherEventWithType:location:modifierFlags:timestamp:windowNumber:context:subtype:data1:data2:` with type=NSSystemDefined(14), subtype=8, data1 = `(NX_KEYTYPE_PLAY << 16) | (keyDown ? 0xA00 : 0xB00)`, data2 = -1; then `CGEventPost(0, event.CGEvent)`. If the objc2 NSEvent surface is awkward, fall back to the documented CGEvent approach. The executor MUST verify the objc2 API surface compiles (RESEARCH A7 — verify).

```rust
use std::sync::atomic::{AtomicBool, Ordering};
static I_PAUSED: AtomicBool = AtomicBool::new(false);

#[cfg(target_os = "macos")]
fn send_play_pause() { /* build + post NSSystemDefined subtype 8, NX_KEYTYPE_PLAY down+up */ }

#[cfg(target_os = "macos")]
pub fn pause_if_playing() {
    // We can't cleanly query "is something playing" without private API.
    // Toggle once and record that WE toggled, so resume can toggle back.
    send_play_pause();
    I_PAUSED.store(true, Ordering::SeqCst);
}
#[cfg(target_os = "macos")]
pub fn resume_if_paused() {
    if I_PAUSED.swap(false, Ordering::SeqCst) { send_play_pause(); }
}
#[cfg(not(target_os = "macos"))]
pub fn pause_if_playing() {}
#[cfg(not(target_os = "macos"))]
pub fn resume_if_paused() {}
```
NOTE on the "only if playing" UX (D-08): a true play-state query needs private MediaRemote (rejected, RESEARCH). The accepted Phase-1 behavior is: toggle on start, toggle back on stop/cancel — symmetric, so if nothing was playing the user gets a no-op pair. Document this tradeoff in the SUMMARY. The flag is gated by `pause_media` in commands.rs (Task 3), so it only runs when the user opted in.
