con default por hardware; `whisper_backend.rs` con threads clamp.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| sidecar `asr` ← bundle | El binario sidecar se localiza por nombre; no se ejecuta en el gate (solo `.is_ok()`) |
| `sw_vers` / `sysctl` ← proceso hijo | Salida de binarios del sistema parseada en el arranque |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | parse de `sw_vers`/`sysctl` output | accept | Binarios del sistema en PATH controlado; parse con fallback seguro (version→0 desactiva Apple, RAM→8 elige small). Sin input de usuario. |
| T-02-02 | Elevation | `sidecar_available` localiza pero no ejecuta el sidecar | mitigate | Usar `.sidecar("asr").is_ok()` (solo resuelve la ruta); la ejecución real sigue gateada por COMPAT-02 + permisos del usuario. |
| T-02-03 | Denial | `available_parallelism()` error | accept | `.unwrap_or(4)` garantiza un valor válido; nunca panic. |
</threat_model>
