<notes>
Hechos clave (de 05-CONTEXT.md, decisión POLISH-01):
- El sidecar `asr-aarch64-apple-darwin` está COMMITEADO en `src-tauri/binaries/`, así que `cargo build` no-bundle NO necesita cross-compile ni reconstruir el sidecar Swift. Por eso el CI de PR es mucho más simple que release.yml: NO necesita los pasos de `swift build`, `xcode-select`, ni dual-triple.
- NO correr `npm run tauri build` en el CI de PR (más lento, requiere firma). Solo `cargo build` para validar que el backend compila.
- `npm run check` = `svelte-kit sync && svelte-check --tsconfig ./tsconfig.json` (verificado en package.json). `svelte-kit sync` necesita que `npm ci` haya corrido antes.
- Clippy/fmt estrictos están DIFERIDOS (05-CONTEXT.md > Deferred Ideas): no añadirlos como gate bloqueante en esta fase, porque hay warnings preexistentes. Si se añade clippy, debe ser no-bloqueante (`continue-on-error: true`) — Claude's Discretion, pero NO romper el build por warnings.
- Runner `macos-15` (consistente con release.yml; necesario porque cargo build linkea frameworks de macOS y el sidecar es Mach-O arm64).
- Cachear cargo (Swatinem/rust-cache@v2 con workspaces: src-tauri) y npm (cache: npm en setup-node) — Claude's Discretion pero recomendado para velocidad.
</notes>
